Why teams need this now

Generative AI has made credible fakes cheap and instant. Traditional fraud tools weren’t built to understand pixels – FakeImage Detector is.

High-stakes decisions

Claims, credit, and compliance teams are increasingly asked to rely on user-submitted photos. When those photos are wrong, the losses are real.

Difficult to detect by eye

Modern AI inpainting tools produce edits that are nearly impossible to spot, even for trained reviewers on large screens.

Need for explainability

A single “real/fake” score doesn’t make a case. Pixel-level overlays give reviewers something they can screenshot, annotate, and share.

How it works

Behind the scenes, FakeImage Detector uses a segmentation model tuned specifically for AI-generated edits, wrapped in a simple web experience.

1. Image ingestion

The image is validated, normalized, and resized so the model can process it reliably, regardless of the original source.

2. AI manipulation detection

A computer vision model scans the image and outputs a per-pixel probability that each region has been synthetically altered.

3. Visual overlay for reviewers

The probabilities are converted into an overlay mask and rendered on top of the original so reviewers can see the suspicious areas instantly.

Technical Architecture

Built as an API-first service that can power both our hosted UI and your internal tooling.

1. Frontend

A lightweight web client handles uploads, displays overlays, and exposes a simple “review & export” workflow for investigators.

2. Detection API

A stateless detection service receives images via REST, runs them through the model, and returns a mask plus metadata in JSON – suitable for integration into claim systems or moderation queues.

3. Model serving

The segmentation model is containerized and deployed behind the API, allowing horizontal scaling as image volume increases and updates without downtime.

Deployment options
  • Hosted: run entirely in our managed environment.
  • Private cloud: deploy the API and model into your own VPC for stricter data controls.
  • API-only: call the detection endpoint from existing case management or moderation tools.
